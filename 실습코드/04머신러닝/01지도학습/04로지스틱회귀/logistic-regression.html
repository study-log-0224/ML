이야기로 시작해봅시다

반 친구 8명의 공부시간과 시험 결과입니다.

그럼 4.5시간 공부한 친구는요? "몰라, 반반쯤?"
이 "반반쯤"을 숫자로 정확히 계산해주는 게 로지스틱 회귀입니다.

왜 직선으로는 안 될까

"공부시간이 늘면 합격률도 오르니까 그냥 직선 그으면 되잖아?"
그런데 직선을 쭉 그으면 이런 일이 벌어집니다.

확률이 150%, −30%가 나옵니다. 말이 안 되죠. 확률은 0%에서 100% 사이여야 하니까요.

그래서 직선을 S자로 구부립니다

직선의 양 끝을 꾹 눌러서 0과 1에 딱 붙게 만듭니다.

하는 일은 딱 하나입니다. 어떤 숫자가 들어와도 0~1 사이로 찌그러뜨리기.

그래서 안심하고 "확률"이라고 부를 수 있게 되죠.

그 S자, 공식은 이렇게 생겼습니다

모양만 봤으니 이제 안을 열어봅시다. 겁먹을 필요 없어요. 줄 하나가 전부입니다.

sigmoid = 1 / (1 + np.exp(-z))

np.exp(-z)는 e의 −z제곱이라는 뜻인데, e가 뭔지는 몰라도 됩니다. 그냥 2.718쯤 되는 고정된 숫자예요. 중요한 건 숫자를 넣으면 뭐가 나오냐입니다.

왜 1을 절대 안 넘을까

np.exp(-z)는 무슨 값을 넣어도 항상 양수입니다. 그러니 분모인 1 + 양수는 무조건 1보다 크죠. 1을 1보다 큰 수로 나누면 반드시 1보다 작습니다. 반대로 양수로 나눴으니 0보다는 크고요. 결과가 0과 1 사이에 갇히는 이유가 이게 전부입니다.

가운데 줄을 꼭 기억하세요. z = 0이면 1 ÷ 2 = 0.5. 계산할 것도 없이 딱 떨어집니다. 그래서 0.5가 판정 기준선이 되는 거예요. 나중에 "합격선이 몇 시간이냐"를 구할 때 이 성질을 씁니다.

이미 20시간 공부한 사람에게 4시간을 더 시키면? 거의 변화 없음. 99% → 99.9%

딱 걸쳐 있는 5시간짜리에게 4시간을 더 시키면? 확 뜁니다. 50% → 88%

잘하는 애한테 과외 더 붙여줘 봤자 별 차이 없고, 애매한 애를 조금 도와주면 확 오르잖아요. 시그모이드는 그 감각을 그대로 수식으로 옮긴 것입니다.

가파름

여기서 놀라운 사실 하나. 셋 다 똑같은 공식입니다. 다른 함수를 쓴 게 아니에요.

모양이 다른 게 아니라 S자의 어느 부분을 보고 있느냐의 차이입니다. 지구는 둥근데 동네에서는 평평해 보이는 것과 같아요. 가운데만 확대하면 직선처럼 보입니다.

모양확률 분포뜻
가파른 S0.01 ~ 0.99대부분을 확신 있게 판정
완만한 S0.12 ~ 0.88절반쯤은 애매함
거의 직선0.27 ~ 0.73전부 "잘 모르겠다"
w는 누가 정할까요? 사람이 아닙니다. model.fit()이 데이터를 보고 찾아내요. 데이터에 신호가 강하면 큰 w가, 약하면 작은 w가 나옵니다. 즉 곡선 모양은 모델이 그리는 게 아니라 데이터가 결정합니다. 곡선이 밋밋하다면 모델을 탓할 게 아니라 변수를 더 찾아야 한다는 신호예요.

단, 급하다고 무조건 좋은 건 아닙니다. 계단처럼 지나치게 가파르면 정답과 거의 같은 변수를 실수로 넣었거나, 과적합이거나, 문제가 원래 너무 쉬운 경우예요. 정확도 99%가 나오면 기뻐하기 전에 이 셋을 먼저 의심하는 게 실무 감각입니다.

양 끝이 눕는 구간의 함정

S자의 양 끝, 그 평평한 부분을 포화 구간이라고 부릅니다. 여기서는 아무리 값을 키워도 확률이 안 변합니다.

눈으로는 0과 1에 닿은 것처럼 보이지만 실제로는 절대 닿지 않아요. z가 8일 때 0.99966, 100일 때 0.9999…, 100만이어도 여전히 1이 아닙니다. 분모가 항상 1보다 크니까요.

실제로 사고가 납니다

모델을 저장했다가 새 데이터로 예측할 때, 스케일러를 빠뜨리면 이런 일이 벌어집니다.

z 값나온 확률
스케일러 O1.420.806
스케일러 X35.540.9999999999

나이 60을 표준화 없이 그대로 넣으면 0.36 × 60 = 21.5가 되어버립니다. 원래는 0.64가 들어가야 할 자리죠. 그래서 z가 35까지 치솟고, 곡선의 오른쪽 끝에서도 한참 벗어난 곳에 떨어집니다.

무서운 건 에러가 안 난다는 점입니다. 그냥 "확률 100%"라는 그럴듯한 답이 조용히 나와요. 모델과 스케일러는 항상 한 쌍으로 저장하고 한 쌍으로 불러와야 합니다.

확률을 답으로 바꾸기

모델이 "합격 확률 78%"라고 하면, 마지막에 기준선을 그어서 자릅니다.

이 50%는 고정이 아닙니다. 상황에 따라 바꿔요.
  • 🏥 암 검사 → 기준을 20%로 낮춤. 조금만 의심스러워도 "더 검사해보자"고 해야 안전하니까
  • 📧 스팸 필터 → 기준을 90%로 올림. 중요한 메일이 스팸함에 처박히면 큰일이니까

코드로 해보기

import numpy as np
from sklearn.linear_model import LogisticRegression

# 공부시간과 결과 (0=불합격, 1=합격)
X = np.array([[1], [2], [3], [4], [5], [6], [7], [8]])
y = np.array([ 0,    0,    0,    0,    1,    1,    1,    1 ])

model = LogisticRegression()
model.fit(X, y)          # ← 이 한 줄이 "공부하는" 부분

# 시간별로 합격 확률 물어보기
for h in [1, 3, 4.5, 6, 8]:
    p = model.predict_proba([[h]])[0][1]
    print(f"{h}시간 공부 → 합격확률 {p:.0%}")
1시간 공부 → 합격확률   6%
3시간 공부 → 합격확률  25%
4.5시간 공부 → 합격확률 50%   ← 딱 반반!
6시간 공부 → 합격확률  81%
8시간 공부 → 합격확률  96%

앞에서 눈으로 "4~5시간 사이가 경계"라고 봤던 걸, 모델이 4.5시간이라고 정확히 짚어줬습니다.

꼭 기억할 것 하나
predict는 "합격!"만 알려주고, predict_proba는 얼마나 확신하는지까지 알려줍니다. 51%로 겨우 합격인지, 99%로 확실한 합격인지는 완전히 다른 정보죠. 실무에선 이쪽을 훨씬 많이 씁니다.

그래프를 받으면 이 순서로 봅니다

실전 체크리스트입니다. 위에서부터 차례로 짚으면 됩니다.

  1. 0.5를 지나는 지점이 어디인가 합격선입니다. -b / w로 계산돼요. "BMI 18 이상이면 위험군"처럼 바로 쓸 수 있는 숫자가 여기서 나옵니다.
  2. 얼마나 가파른가 모델의 확신도이자 변수의 영향력. 완만하면 이 변수만으로는 부족하다는 뜻입니다.
  3. 어느 방향인가 오른쪽 위로 가면 값이 클수록 1에 가까워지고(공부시간), 오른쪽 아래면 반대입니다(결석일수). 상식과 어긋나면 데이터를 의심하세요.
  4. 양 끝이 눕는가 포화 구간입니다. 거기 있는 샘플은 더 투자해도 효과가 없어요. 어디에 자원을 넣어야 할지 알려줍니다.
  5. 실제 데이터 점이 어떻게 흩어져 있는가 색이 좌우로 깔끔히 갈리면 좋은 모델, 뒤섞여 있으면 변수를 더 찾아야 합니다.

한 줄로 줄이면 — "어디서 갈리는가(위치)"와 "얼마나 확실히 갈리는가(기울기)". 시그모이드 그래프는 이 둘을 동시에 보여주는 그림입니다.

진짜 매력 — 이유를 말해줍니다

여러 정보를 같이 넣으면, 모델이 뭐가 중요했는지 숫자로 알려줍니다.

이게 왜 중요하냐면 — 요즘 유행하는 딥러닝은 성능은 좋은데 "왜 그렇게 판단했는지" 설명을 못 합니다. 은행에서 대출을 거절당했는데 "AI가 안 된대요"라고만 하면 곤란하잖아요. 그래서 이유를 설명해야 하는 분야(은행, 병원)에서는 로지스틱 회귀가 아직도 현역입니다.

개 / 고양이 / 새처럼 셋 이상을 나누고 싶다면? y에 0, 1, 2를 넣기만 하면 사이킷런이 알아서 처리해줍니다. 각각의 확률(개 70%, 고양이 25%, 새 5%)을 계산해서 제일 높은 걸 고르는 방식이에요.
