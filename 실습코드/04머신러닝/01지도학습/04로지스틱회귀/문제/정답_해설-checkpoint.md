## 이 과제의 진짜 함정은 01단계에 있다

**의학의 "양성"과 머신러닝의 "양성"은 뜻이 정반대다.**

| 단어 | 의학에서 | 머신러닝에서 |
|---|---|---|
| **양성** | benign — 순한 혹, **괜찮다** | positive — **찾고 싶은 쪽**, 라벨 1 |

이 데이터에서 `target=1`은 **benign(순한 혹)**입니다. 즉 sklearn이 말하는 "양성 클래스(1)"가 **건강한 쪽**입니다. 우리가 정작 찾고 싶은 건 **악성(0)**인데 말이죠.

```python
print('target_names:', list(data.target_names))   # ['malignant', 'benign']
```

**02단계에서 이 한 줄을 안 찍으면 끝까지 뒤집어 읽습니다.** recall이 높다고 좋아했는데 "건강한 사람을 잘 찾았다"는 뜻일 수 있죠.

이게 코드로도 나타납니다 — 08단계에서 악성의 recall을 구하려면 `recall_score(..., pos_label=0)`으로 **sklearn의 기본값을 뒤집어** 줘야 합니다.

## 실행 결과 (노트북 실행 실측값)

```
X: (569, 30)
target   1: 357 (양성 benign) / 0: 212 (악성 malignant)
분할: (455, 30) (114, 30)

정확도 : 0.9825
ROC-AUC: 0.9957
```

본편은 AUC 0.63이었는데 여기는 **0.9957**입니다. **같은 로지스틱, 같은 코드**인데요. 모델이 똑똑해진 게 아니라 **데이터가 다른 것**입니다 — 세포 측정치는 악성/양성을 실제로 잘 가르는 정보를 담고 있습니다.

## 정확도 0.98에 만족하고 넘어가면 안 된다

```
[[40  2]
 [ 0 72]]

악성인데 양성이라 놓친 수: 2
양성인데 악성이라 오진한 수: 0
```

**하필 "악성을 놓친 쪽"으로만 2건 틀렸습니다.** 반대 방향 실수는 0건입니다.

| 실수 | 결과 |
|---|---|
| 악성을 놓침 (FN) | 환자가 **치료를 못 받고 병이 진행**된다 |
| 양성을 악성이라 함 (FP) | 조직검사를 한 번 더 한다. **놀라지만 살 수 있다** |

정확도 98.25%는 훌륭해 보이지만, **그 1.75%가 전부 최악의 방향**입니다. 지표 하나만 보면 이걸 놓칩니다.

## 문제 4번 답 — 어느 실수가 더 위험한가

**"악성을 놓치는 것(FN)"이 훨씬 위험합니다.** 그래서 **악성의 재현율(recall)**을 중시해야 합니다. 말로 끝내지 말고 임계값으로 답해 봅시다.

```
임계값 0.5 -> 놓친 악성 2명 | 정확도 0.9825 | 악성 recall 0.9524
임계값 0.9 -> 놓친 악성 1명 | 정확도 0.9386 | 악성 recall 0.9762
```

임계값 0.9는 **"양성(건강)이라고 말하려면 90% 이상 확신해야 한다"** = 조금이라도 의심스러우면 악성으로 분류한다는 뜻입니다.

**정확도를 4%p 내주고 환자 한 명을 살렸습니다.** 남는 장사일까요? 의료에서는 그렇습니다 — **정확도는 숫자고 환자는 사람입니다.**

**지표는 목적이 아니라 도구입니다.** 01단계에서 "무엇이 중요한가"를 정하지 않으면 정확도가 높은 쪽을 고르게 되고, **그게 사람을 잡습니다.**

## 본편과 다른 결과 — 여기선 스케일링이 성능에도 영향을 준다

```
[스케일링 O] 정확도 0.9825 | AUC 0.9957
[스케일링 X] 정확도 0.9474 | AUC 0.9924
```

본편(health_survey)에서는 스케일링을 빼도 정확도가 **0.6150으로 똑같았습니다.** 여기서는 **0.9825 → 0.9474**로 확실히 떨어집니다. 왜 다를까요?

**로지스틱에는 기본으로 규제(C=1.0)가 걸려 있습니다.** 규제는 "계수를 너무 크게 만들지 마라"라고 누르는 장치죠. 그런데 스케일을 안 맞추면 **이 누르는 힘이 특성마다 불공평하게 걸립니다.**

| 특성 | 범위 | 규제의 영향 |
|---|---|---|
| `mean area` | 143 ~ 2501 | 계수가 작아도 되니 **규제를 거의 안 받는다** |
| `mean smoothness` | 0.05 ~ 0.16 | 계수가 커야 하는데 **규제에 눌려 죽는다** |

즉 **규제가 "범위가 작은 특성"만 골라서 죽입니다.**

본편은 특성 4개에 범위 차이 40배라 이 왜곡이 눈에 안 띄었습니다. 여기는 **특성 30개에 범위 차이 15,000배**라 성능으로 드러난 것입니다.

**교훈: "스케일링은 성능과 무관하다"는 절대 규칙이 아닙니다.** 본편에서 그렇게 보였을 뿐이죠. 범위 차이가 크고 특성이 많으면 성능에도 영향이 옵니다. **그러니 로지스틱에는 그냥 항상 하는 게 맞습니다.**
